class src.pyLIQTR.BlockEncodings.DoubleFactorized.DoubleFactorized(ProblemInstance, df_error_threshold: float = 0.001, sf_error_threshold=1e-08, br: int = 7, phase_gradient_eps=1e-10, energy_error=0.001, step_error=None, bits_rot_givens=None, keep_bitsize=None, outer_prep_eps=None, **kwargs)[source]

Bases: BlockEncoding

Implements encoding from Appendix C Figure 16 of Ref [1].

References: [1] https://arxiv.org/abs/2011.03494 [2] https://arxiv.org/abs/2007.14460

Parameters:

  • ProblemInstance (ProblemInstance) – A pyLIQTR.ProblemInstance for the system of interest. Currently supports ChemicalHamiltonian instances.

  • df_error_threshold (float) – The threshold used to throw out factors from the double factorization. Truncation is carried out as described in Appendix C Section 3 of Ref [1]. This parameter corresponds to the RHS of Eq C41.

  • sf_error_threshold (float) – The threshold used to throw out factors from the first eigendecomposition. Terms with eigenvalues less than or equal to sf_error_threshold are thrown out. This decreases L, the rank of the two body tensor in eq C10.

  • br (int) – The number of precision bits to use for the rotation angles output by the QROM in step 2 and used in step 3a)iv on page 52 of Ref [1]. Note, this is not used in step 1a which does not currently use a phase gradient rotation.

  • phase_gradient_eps (float) – Overall error in gradient state preparation, ie each rotation done to prepare the phase gradient state will be performed with error phase_gradient_eps/bits_rot_givens

  • energy_error (float) – The allowable error in phase estimation energy. Used to set bits_rot_givens, keep_bitsize, and outer_prep_eps if step_error is not passed as input.

Optional inputs:

Parameters:

  • step_error (float) – Error in spectral norm of walk operator. Corresponds to epsilon in eq C24 of Ref [1]. Used to set bits_rot_givens, keep_bitsize, and outer_prep_eps if those are not passed as inputs.

  • bits_rot_givens (int) – The number of precision bits to use for the Givens rotations in step 4 on page 53 of Ref [1]. Called $eth$ in Ref [1].

  • keep_bitsize (innt) – Number of precision bits to use for preparing the coefficients on the second selection register (p). N2 in Eq C31 of Ref [1].

  • outer_prep_eps (float) – Precision of normalized coefficients prepared on first selection register (l). Used to set N1 in Eq C27 of Ref [1].

an instance of the DoubleFactorized() class

class src.pyLIQTR.BlockEncodings.FirstQuantized.FirstQuantized(ProblemInstance, geometry_file: str = None, num_bits_rot_aa: int = 8, eps_M: float = 0.0001, eps_R: float = 0.0001, eps_T: float = 0.0001, n_M=None, n_T=None, n_R=None, **kwargs)[source]

Bases: BlockEncoding_select_prepare

Implements encoding from Section II of the reference.

References

Fault-Tolerant Quantum Simulations of Chemistry in First Quantization

Parameters:

  • ProblemInstance (ProblemInstance) – A pyLIQTR.ProblemInstance for the system of interest. Currently supports MoleculeParameters and ElectronicStructure instances.

  • geometry_file (str) – File containing the atomic coordinates for the system of interest. Only needed if ProblemInstance is ElectronicStructure.

  • num_bit_rot_aa (int) – The number of precision bits used for rotating the ancilla qubit used for amplitude amplification in the uniform superpostion subroutine, equivalent to \(b_r\) in the reference.

  • eps_M (float) – The allowable error in the estimated energy due to imprecise preparation of the state with amplitudes \(1/||\nu||\)

  • eps_R (float) – The allowable error in energies due to the finite precision of the atomic positions.

  • eps_T (float) – Allowable error in energies due to the finite precision in the rotation of the qubit selecting between T and U + V.

  • n_M (int) – The number of bits used in the inequality testing to prepare a state with amplitudes \(1/||\nu||\), equivalent to \(n_{\mathcal{M}}\) in the reference.

  • n_T (int) – The number of bits used for rotating the ancilla qubit used to select between T and U+V, equivalent to \(n_T\) in the reference.

  • n_R (int) – The number of bits used to store each component of the atomic position, equivalent to \(n_R\) in the reference.
