class src.pyLIQTR.ProblemInstances.ElectronicStructure.ElectronicStructure(filenameH: str, filenameG: str, **kwargs)[source]

Bases: ProblemInstance

This ProblemInstance represents periodic electronic structure Hamiltonians generated using PEST. The Hamiltonians generally take on the form

\[H = \sum_{p,q,\sigma} T(p-q) a^\dagger_{p,\sigma}a_{q,\sigma} + \sum_{p,\sigma} U(p) n_{p,\sigma} + \sum_{(p,\alpha)\neq(q,\beta)} V(p-q)n_{p,\alpha}n_{q,\beta}\]

where \(a^\dagger_{p,\sigma}\) and \(a_{p,\sigma}\) are fermionic creation and annihilation operators on spatial orbital \(p\) with spin \(\sigma\) in {up,down}, and \(n_{p,\sigma} = a^\dagger_{p,\sigma}a_{p,\sigma}\) is the number operator.

Given the PEST files, this ProblemInstance can yield the information needed to encode the Hamiltonian using the LinearT, PauliLCU or FirstQuantized block encoding. It also provides descriptions of the electronic structure system size with M_vals, the number of grid points along each spatial dimension, and N, the total number of orbitals.

Parameters:

  • filenameH (str) – filename for PEST generated ham.hdf5 file containing the Hamiltonian coefficients

  • filenameG (str) – filename for PEST generated grid.hdf5 file containing the basis grid

class src.pyLIQTR.ProblemInstances.MoleculeParameters.MoleculeParameters(cell_volume: float, num_electrons: int | None = None, num_atoms: int | None = None, sum_atomic_num: int | None = None, geometry_file: str | None = None, num_plane_waves: int | None = None, E_cut_off: float | None = None, lengths: list | None = None, **kwargs)[source]

Bases: ProblemInstance

This ProblemInstance captures the information about a molecular system that is needed for the FirstQuantized block encoding. The parameters ultimately needed are cell_volume, num_electrons, num_atoms, sum_atomic_num and num_plane_waves. A geometry file (.xyz) for the system of interest can be optionally used to set num_electrons, num_atoms, and sum_atomic_num, by passing it in with geometry_file=. Additionally, the number of plane waves (num_plane_waves) can be optionally calculated from the desired E_cut_off and lengths.

Parameters:

  • cell_volume (float) – The computational cell volume needed for the physical system of interest.

  • num_electrons (Optional[int]) – The number of electrons in the system.

  • num_atoms (Optional[int]) – The number of atoms in the system.

  • sum_atomic_num (Optional[int]) – The sum of the atomic numbers.

  • geometry_file (Optional[str]) – The path to the .xyz file containing information about the atoms in the system and their coordinates. This file can be used to set num_electrons, num_atoms, and sum_atomic_num.

  • num_plane_waves (Optional[int]) – The number of plane waves the make up the system basis.

  • E_cut_off (Optional[float]) – The converged energy cut off in Rydberg units. This can be used together with lengths to set num_plane_waves.

  • lengths (Optional[list]) – A list containing the length of each dimension of the computational cell. This can be used together with E_cut_off to set num_plane_waves.

class src.pyLIQTR.ProblemInstances.fermionic_models.FermiHubbardNNN(shape=(4, 4), J1=-1.0, J2=-1.0, U=4.0, pbcs=(False, False), **kwargs)[source]

Bases: ProblemInstance

This ProblemInstance represents the Fermi-Hubbard model including nearest and next-nearest neighbors terms. The corresponding Hamiltonian can be written as

\[-t\sum_{\langle p,q\rangle,\sigma} a^\dagger_{p,\sigma} a_{q,\sigma}-t'\sum_{\langle p,q\rangle',\sigma} a^\dagger_{p,\sigma} a_{q,\sigma} + \frac{u}{2}\sum_{p,\alpha\neq\beta}n_{p,\alpha}n_{p,\beta}\]

where the first sum is over nearest neighbors and the second over next-nearest.

Parameters:

  • shape (Tuple[int]) – A Tuple containing the dimensions of the lattice. Note, currently only square 2D lattices are supported.

  • J1 (float) – The nearest neghbor hopping coefficient. Represented as \(t\) in the above equation.

  • J2 (float) – The next-nearest neghbor hopping coefficient. Represented as \(t'\) in the above equation.

  • U (float) – Onsite interaction strength. Represented as \(u\) in the above equation.

  • pbcs (Tuple[bool]) – Determines if periodic boundary conditions are on (True) or off (False) for the corresponding dimension. This will affect which terms are included when using the PauliLCU block encoding. Note that the LinearT block encoding assumes periodic boundary conditions.
